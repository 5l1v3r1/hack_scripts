
Каждый путь или несколько путей теперь связаны с одним или несколькими тегами.

Пример:

!define PATH `'{"[c:]"} {"$PROGRAMFILES32" "[$APPDATA]"}'`

!define WORD `'{*pagtefile.sys} {*utorrent *microsoft}'`

Здесь путь {"[c:]"} связан только с тегом {*pagtefile.sys} а два пути {"$PROGRAMFILES32" "[$APPDATA]"} связаны только с двумя тегами {*utorrent *microsoft}

[] - Путь зкалюченный в квадратные скобки указывает скрипту что подпапки сканировать ненужно.
"" - Двойные кавычки теперь обязательны для всех путей. Это помогает избежать проблем которые возникают если путь содержит пробелы или если переменная например $PROGRAMFILES32 при раскрытии содержит пробелы c:\program files.

{} - Фигурные скобки используються для того что бы группе путей задавать в соответствие группу тегов.

Например:

!define PATH `'{"c:"}`

!define WORD `'{*folder1 folder2}'`

Таким образом два тега folder1 и folder2 связаны только с путем указывающим на диск C:

Путей и тегов в связке может быть сколько угодно. Например:

!define PATH `'{"c:" "$PROGRAMFILES32" "$APPDATA"}`

!define WORD `'{*folder1 folder2}'`

Деление на группы обязательно. Если группы ненужны то значит нужно создать одну для всего и все пути заключит разом в одни фигурные скобки и тоже самое толжно быть с тегами.

По возможности лутше использовать скриптовые переменные окружения начинающиеся со знака бакса. Можно и виндовые впринцепе это тоже работает но внутренние надёжнее.

Порядок постановки скобок и кавычек только такой:

[] - Квадратные скобки если они есть то всегда внутри. За ними следом двоные кавычки "" если они есть и только потом фигурные скобки.
